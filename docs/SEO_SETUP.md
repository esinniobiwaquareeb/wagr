# SEO Setup Guide for wagered.app

This document outlines the SEO improvements implemented and how to maintain them.

## Implemented Features

### 1. Structured Data (Schema.org)
- **Organization Schema**: Company information, logo, contact details
- **WebSite Schema**: Site search functionality
- **SiteNavigationElement**: Navigation links (About, Contact, FAQ, etc.)
- **FAQPage Schema**: FAQ questions and answers for rich snippets
- **ContactPage Schema**: Contact page information
- **AboutPage Schema**: About page information
- **BreadcrumbList Schema**: Breadcrumb navigation for better site structure

### 2. Breadcrumbs
- Visual breadcrumb navigation on key pages
- Structured data for breadcrumbs
- Accessible navigation with ARIA labels

### 3. Enhanced Metadata
- Page-specific metadata for all public pages
- Open Graph tags for social sharing
- Twitter Card metadata
- Proper title and description tags

### 4. Sitemap
- Comprehensive sitemap with all important pages
- Proper priorities and change frequencies
- Automatically generated by Next.js

### 5. robots.txt
- Properly configured to allow important pages
- Blocks admin and API routes
- Includes sitemap reference

## Next Steps for Google Search Console

### 1. Submit Sitemap
1. Go to [Google Search Console](https://search.google.com/search-console)
2. Add your property (wagered.app)
3. Verify ownership (DNS, HTML file, or meta tag)
4. Navigate to Sitemaps section
5. Submit: `https://wagered.app/sitemap.xml`

### 2. Add Social Media Links
When you have social media accounts, update the Organization schema in:
- `wagr/components/seo/structured-data.tsx`
- Add URLs to the `sameAs` array:
  ```typescript
  "sameAs": [
    "https://twitter.com/wageredapp",
    "https://facebook.com/wageredapp",
    "https://instagram.com/wageredapp",
    "https://linkedin.com/company/wageredapp"
  ]
  ```

### 3. Monitor Performance
- Check Search Console for indexing status
- Monitor rich snippet appearance
- Track click-through rates
- Review search queries

### 4. Additional Recommendations

#### Add Article Schema (if you add blog)
If you add a blog section, implement Article schema:
```typescript
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Article Title",
  "author": {
    "@type": "Organization",
    "name": "wagered.app"
  },
  "datePublished": "2024-01-01",
  "dateModified": "2024-01-01"
}
```

#### Add Review/Rating Schema (if applicable)
If you collect user reviews:
```typescript
{
  "@context": "https://schema.org",
  "@type": "Review",
  "reviewRating": {
    "@type": "Rating",
    "ratingValue": "5",
    "bestRating": "5"
  },
  "author": {
    "@type": "Person",
    "name": "User Name"
  }
}
```

## Testing

### Validate Structured Data
1. Use [Google's Rich Results Test](https://search.google.com/test/rich-results)
2. Enter your page URL
3. Check for any errors or warnings

### Test Breadcrumbs
1. Use [Schema.org Validator](https://validator.schema.org/)
2. Check breadcrumb navigation visually on pages
3. Verify accessibility with screen readers

### Check Sitemap
1. Visit `https://wagered.app/sitemap.xml`
2. Verify all pages are listed
3. Check lastModified dates

## Maintenance

### Regular Updates
- Update sitemap priorities as content changes
- Keep metadata descriptions fresh
- Add new pages to sitemap
- Update breadcrumbs for new page structures

### Monitoring
- Check Google Search Console weekly
- Monitor indexing status
- Review search performance
- Fix any structured data errors
